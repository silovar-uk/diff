# Text Review Studio v0.6.6

左右比較を中心に、校正・校閲・CMS整形までを**ブラウザ内だけ**で扱う静的Webアプリです。GitHub Pagesにそのまま置けます。

v0.6.6は、CMSのHTMLタグに振り回されず、本文の変更をdifff型の左右比較で確認できるようにした版です。

## v0.6.6の更新

### CMSタグを無視して本文を比較

- `左右で比較`を開くと、上部に**タグを無視**トグルを表示します。初期状態はONです。
- `<span>`、`<table>`、`<img>`のように、`<>`の中が英字で始まるHTMLタグを比較対象から外します。
- タグ・class・styleなどの差分に引っ張られず、見出しや本文、表記だけを左右で確認できます。
- トグルをOFFにすると、HTMLタグも含めて比較できます。
- `x < 3`のような通常の比較記号はタグとして扱いません。

### difff型の左右差分

- 比較は**行単位で対応付けてから、同じ行の中を文字単位で比較**する方式に変更しました。
- 変更のある行は左右で同じ高さに並び、変更文字だけを左は文字色、右はマーカーで示します。
- これまでのように複数行の変更が大きな一塊になり、本文・タグ・属性の差が混ざる挙動を抑えています。

## v0.6.5の更新

- 編集中のCMS作業版でも、検索結果すべてに淡い青のハイライトを表示。現在位置はブラウザの文字選択に近い濃いハイライトで示します。
- タグ追加時に、編集欄のスクロール位置と選択位置を保持。画面上部へ戻らず、その場で作業を続けられます。
- 検索に**正規表現モード**を追加。例：`、$`で各行の末尾にある読点を検索できます。`/pattern/i`形式も使えます。
- 左側と検索バーの両方に、正規表現の例として「文末の、」を追加。

## v0.6.4の更新

- 左側の「探す」に、全角の開きかっこ `（` と閉じかっこ `）` を個別に探せるクイック検索を追加

## v0.6.3の更新

### 初回導線

- 空の状態で、左に**変更前**、右に**修正後**を貼る順番を番号付きで案内
- 片側だけ入力した場合は、次に貼る場所を明示
- 右だけ入力した場合は、比較なしでCMS作業を始められることを案内
- 右側の原稿が空の間はコピーを無効化

### 差分の見やすさ

- **変更前（左）**：変更・削除された文字はコーラル系の**文字色のみ**
  - 背景マーカーなし
  - 下線なし
  - 取消線なし
  - 行全体のハイライトなし
- **修正後（右）**：追加・置換された文字だけをグリーン系の**背景マーカー**で表示
- 現在確認している差分は、本文ではなく中央ガターの `↔ / + / −` で示す
- 初回比較時に、赤い文字・緑マーカー・中央記号の意味を説明する凡例を表示

### 左側の常設メニュー

操作は常に左側に残しますが、初見の圧迫感を減らすため3つに整理しました。

1. **まず使う**：検索、全角空白、`! ?`の半角化、表記統一、空行整理
2. **こだわり**：適用中の表記セット、厳密モード、例外ルール
3. **くわしい操作**：タグ付け、HTMLの扱い、表示レイヤー

画面幅に応じて、左メニューは 240px → 192px → アイコンレールへ縮小します。本文の可読幅を優先します。

### 確認レール

- 右端は縦書きではなく、`確認 / 5件 / 重要 2` のように横組みで表示
- 初回だけ「ここに確認が必要な項目が集まります」と案内
- 差分、表記、校閲、HTMLで**判断が必要な項目だけ**を扱います

## 基本操作

1. 左の**変更前**に比較元の原稿を貼り付けます。
2. 右の**修正後**に修正版を貼り付けます。
3. `左右で比較` を押すと、段落・行を対応させた差分ビューになります。
4. 左メニューから検索、整形、タグ付けを行います。
5. 右端の`確認`で、未確認の差分・表記・校閲・HTML項目を処理します。
6. 右上の`コピー`から、CMS本文向けテキスト、HTML、確認記録をコピーします。

## こだわりセット

- `浦和公式サイト｜お知らせ記事`
- `CMS作業｜HTML優先`
- `自由編集｜提案を控えめ`

ルールは次の3段階で扱います。

- **絶対守る**：URL、メール、HTMLなどを保護
- **原則守る**：空白、記号、統一表記
- **文脈で判断**：見出し、引用、固有表記

候補ごとに、`反映`、`今回は残す`、`今後もこの表記を残す`を選べます。最後の選択は、例外として保存されます。

## ローカル処理とデータ

- 入力内容は外部APIに送信しません。
- 外部CDN、分析タグ、外部フォントは使いません。
- 作業内容、こだわりセット、例外ルールはブラウザの`localStorage`に保存されます。
- v0.6.0〜v0.6.5の保存データがあれば、v0.6.6で引き継ぎを試みます。
- 重要原稿は`… > 作業データを書き出す`からJSONでも控えてください。

共有PCでは、作業後に`… > この画面の内容を消去`を使ってください。

## GitHub Pages公開手順

1. ZIPを解凍します。
2. フォルダ内のファイルをGitHubリポジトリ直下へ置きます。
3. GitHubの **Settings > Pages** を開きます。
4. `Deploy from a branch` を選びます。
5. Branchは `main`、Folderは `/(root)` を選んで保存します。
6. 数分後、表示されたPages URLで開きます。

ビルド不要の静的ファイル構成です。

## ショートカット

- `Ctrl / Cmd + K`：操作パレット
- `← / →`：前後の確認項目へ移動
- `A`：選択中の項目を反映／確認済み
- `S`：選択中の項目を保留
- `Ctrl / Cmd + Z`：元に戻す
- `Ctrl / Cmd + Shift + Z`：やり直す
