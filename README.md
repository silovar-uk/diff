# Text Review Studio v0.4.0

原稿を読みながら、差分・表記統一・HTMLラベル化を**1画面で確認し、そのままコピーできる**ローカル完結型Webアプリです。

入力内容を外部APIへ送信しません。外部フォント、CDN、解析タグも使用していません。

## v0.4.0で変わったこと

- 5ステップ画面を廃止し、本文＋右側の確認キューへ集約
- 差分ロジックを二段階化
  - まず段落・行で比較
  - 変わったブロックだけを文字単位で比較
- 右側の重複表示につながる共有トークン処理を修正
- `【ラベル】` を、候補ごとに以下から選択可能
  - タグにする
  - かっこを外す
  - そのまま残す
- 原稿内にすでにあるHTMLタグも、掲載用HTMLで以下から選択可能
  - 保持する
  - タグだけを外して文字を残す
- タグなしテキスト／掲載用HTML／差分一覧を常設コピー
- 最終監査用の全体比較をオーバーレイで表示

## 使い方

1. 中央の「原稿」に本文を貼り付けます。
2. 必要な場合だけ、下部の「原文を追加して比較」から修正前原稿を入力します。
3. 右側の「確認すること」で、未確認・重要・完了を切り替えます。
4. 候補を選びます。
   - 手動差分：確認済みにする
   - 表記統一：採用／除外
   - `【ラベル】`：タグにする／かっこを外す／そのまま
5. 右上の「コピー」から、用途に合う形式をコピーします。

## タグの扱い

右側の「タグの扱い」を開くと、出力方針を切り替えられます。

### 既存HTMLタグ

- **保持**：原稿内の `<strong>` や `<span>` などを掲載用HTMLに残します。
- **タグを外す**：タグそのものだけを削除し、中の文字は残します。

### `【ラベル】`

初期方針を以下から選べます。個別候補で上書きもできます。

- **タグにする**：`<span class="info24-label">ラベル</span>` に変換
- **かっこを外す**：`ラベル` に変換
- **そのまま**：`【ラベル】` のまま残す

## 出力の種類

- **タグなしテキスト**：既存HTMLタグを除去したプレーンテキスト
- **掲載用HTML**：右側のタグ設定とラベル判断を反映したHTML
- **差分一覧**：上長・制作会社との確認共有用テキスト

```text

styles.css           スタイル
manifest.webmanifest PWA用設定
README.md            このファイル
tests/diff-tests.js  差分ロジックの回帰テスト
```

## GitHub Pagesで公開する

1. ZIPを解凍します。
2. **解凍後の中身をそのまま**GitHubリポジトリ直下へ置きます。
3. GitHubで `Settings` → `Pages` を開きます。
4. `Build and deployment` を `Deploy from a branch` に設定します。
5. Branchを `main`、Folderを `/ (root)` にして保存します。
6. 数分後、表示されたURLで公開されます。

## ローカルでテストする

差分ロジックの回帰テストはNode.jsで実行できます。

```bash
node tests/diff-tests.js
```

ブラウザ確認は、静的サーバーでの実行を推奨します。

```bash
python3 -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## 注意

- HTML構造プレビューは、本番CMSや本番サイトのCSS・許可タグ・改行処理を完全に再現するものではありません。
- 表記統一は候補提示を主目的としています。固有名詞、引用、契約・法務表現、正式名称は必ず人が確認してください。
