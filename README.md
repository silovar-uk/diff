# Text Review Studio v0.5.0

左右比較を核に、校正・校閲・CMS整形までをブラウザ内だけで行う静的Webアプリです。

## 主な機能

- `変更前` と `CMS作業版` を左右に置く差分比較
- 段落・行を先に比較し、変更ブロックだけを細かく表示
- 右端の確認レール
  - 手動差分
  - 表記統一候補
  - 日付・曜日の整合チェック
  - `【ラベル】` のHTML化候補
- CMS作業版の検索・置換
  - 全角／半角スペース、記号、`【】`、URL、HTMLタグの検索チップ
  - URL・メール・HTMLタグを保護した置換
- CMS整形コマンド
  - 空白、記号、表記、改行をプレビューしてから反映
  - `【】` のラベル化、URLリンク化、※注記のタグ化
  - 選択範囲へのラベル／見出し／注意／強調／リンク付与
- 出力設定
  - 既存タグを保持／タグだけ外す
  - `【ラベル】` をタグ化／かっこを外す／残す
  - 改行を維持／`<br>`／段落化
  - URLを文字列のまま／リンク化
- コピー先ベースの出力メニュー
  - CMS本文へ：整形後テキスト
  - CMS HTML欄へ：掲載用HTML
  - 確認依頼へ：差分・確認記録
- CMS加工履歴、Undo / Redo、JSON書き出し・読み込み

## ローカル処理

入力した文章は外部APIへ送信しません。処理と保存はブラウザ内で完結します。

- 外部CDN、分析タグ、外部フォントは使用していません。
- 作業データはブラウザの `localStorage` に自動保存されます。
- 共有PCでは、作業終了後に `… > この画面の内容を消去` を実行してください。
- 念のため、重要な原稿は `… > 作業データを書き出す` からJSONで控えてください。

## GitHub Pages公開手順

1. ZIPを解凍する。
2. フォルダ内のファイルをGitHubリポジトリ直下へ配置する。
3. GitHubの **Settings > Pages** を開く。
4. `Deploy from a branch` を選択する。
5. Branchは `main`、Folderは `/(root)` を選んで保存する。
6. 数分後にPages URLで開く。

このアプリはビルド不要の静的ファイル構成です。

## 操作ショートカット

- `⌘ / Ctrl + K`：操作パレット
- `← / →`：前後の確認項目へ移動
- `A`：選択中の項目を反映／確認済み
- `S`：選択中の項目を保留
- `⌘ / Ctrl + Z`：元に戻す
- `⌘ / Ctrl + Shift + Z`：やり直す
